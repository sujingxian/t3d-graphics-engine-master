◆ ~Renderer()

T3D::Renderer::~Renderer ( void  )
virtual

Destructor NOTE(Evan): Textures are loaded and owned by the callee. Textures do not properly clean up resources on destruction. This means resource buffers and IDs will leak even if they are are stored in a RAII STL container, e.g. std::vector, One way to fix this without implementing an some sort of asset manager is have Textures register themselves with e.g. GLRenderer upon initialisation, which manages their internal buffers, IDs, mip levels etc.
